## 1. Why Domain Knowledge Matters

A Data Engineer must understand:
- What data represents
- Why it exists
- How business uses it

Without this:
- Pipelines will be incorrect
- Debugging becomes difficult
- Interviews become weak

## 4. Real-World Problems

- Sensor data may be delayed
- Missing values in production data
- Different teams define metrics differently
- Data inconsistency across systems

---

## 5. Real Project Thinking

A senior engineer thinks:

- Can this data be trusted?
- What happens if data is missing?
- How do we validate this data?
- How will downstream systems use this?

Example:
If water_cut increases suddenly:
- Is it real?
- Sensor issue?
- Data ingestion delay?
