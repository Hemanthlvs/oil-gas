## 1. What is Enterprise Data Architecture?

Enterprise architecture defines:
- How data flows across systems
- How data is stored and processed
- How different components interact

It ensures:
- Scalability
- Reliability
- Maintainability

---

## 2. Our Target Architecture (Oil & Gas Platform)

We will design a system with the following layers:

### 2.1 Data Sources Layer

Sources include:
- IoT Sensors (streaming)
- ERP Systems (batch)
- Maintenance Logs
- External APIs (weather, oil prices)

---

### 2.2 Ingestion Layer

Responsible for collecting data:
- Batch ingestion → scheduled loads
- Streaming ingestion → real-time events
- API ingestion → external data

---

### 2.3 Storage Layer (Data Lake)

Using Azure Data Lake:

Data is stored in:
- Bronze (raw)
- Silver (cleaned)
- Gold (business-ready)

---

### 2.4 Processing Layer

Using Databricks:
- PySpark transformations
- SQL transformations
- Delta Lake tables

---

### 2.5 Data Quality Layer

Ensures:
- Valid data
- Error handling
- Data validation

---

### 2.6 Governance Layer

Controls:
- Access permissions
- Data lineage
- Data security

---

### 2.7 Serving Layer

Used by:
- Analysts
- Dashboards
- ML models

---

### 2.8 Orchestration Layer

Controls:
- Scheduling
- Workflow execution
- Dependency management

---

## 3. Medallion Architecture (Core Concept)

### Bronze
- Raw data
- No transformations

### Silver
- Cleaned data
- Validated

### Gold
- Aggregated data
- Business KPIs

---

## 4. Example Data Flow

Sensor Data → Ingestion → Bronze → Silver → Gold → Dashboard

---

## 5. Real-World Problems

- Data arriving late
- Pipeline failures
- Schema changes
- High data volume

---

## 6. Real Project Thinking

A senior engineer designs for:

- Failure handling
- Scalability
- Reprocessing capability

Example:
"What if pipeline fails in middle?"
→ Need retry + idempotency
